Icons

There has been a lot of discussion in the industry about the best way of implementing icons on a web page. Icon fonts, such as Font Awesome, are widely available and generally free to use, but have a high overhead and can be tricky to style efficiently.

We've decided to use raw SVG vectors, and include a minimal set within the design system.

Please note: this component will not work on IE8 and below, since it depends on SVG.

Labels

Before we get started, all icons in the sytem require a label to supplement the graphic with a clear definition of what it is, and also act as a fallback on devices and browsers that do not support SVG. By itself, what an icon represents can be a very subjective experience!

Usage

Add the data-icon attribute to any empty element and give it the value of the icon you wish to use, eg. 

<div data-icon="maps">Map</div>
<span data-icon="trash">Trash</span>

This will be replaced at load time with an svg DOM element referencing the desired icon.

By default, the icon will inherit the currentColor value of its parent element, and will be constrained to 60px × 60px.

There are three CSS helper classes that can be used to adjust the size:

  • small 30px × 30px
  • large 100px × 100px
  • fill 100% of container width and height

All text within the element will be kept as a label (try and keep it short). All labels are centred on their containing element.
